Invoice Reader



Problem definition

Invoices come in a wide range of formats, each with its own little characteristic that makes it unique. The process of digitising them manually requries many hours of labour, with employees needing to repeat the same menial task for hours on end. Not only is this expensive and mind-numbing, but the very essence of manual data entry means that the finished product will be error prone with high turnaround times.


Implementation

I aimed to solve this problem working with Linktech using LLMs, OCR, and robust algorithms. Using my own finetuned LLM which was trained off real samples of data, it was able to correctly identify fields of an invoice along with any nuances, and output the invoice as structured data. I complemented this with OCR and designed algorithms to ensure the accuracy of the processed data.


Tools

I published this service as a website with full CRUD functionality, using PostgreSQL as the database. I developed the database structure to allow for intuitive management of invoices for the user, where they are able to manage 'Jobs', 'Documents', 'Invoices', 'Suppliers' etc. Using Django as the framework with HTML, CSS and Javascript, I developed the website to have an interface that integrates seamlessly with the backend.


Hosting

To host this, I made this service into a docker image, which I upload to AWS ECR and use ECS to run the container. This allows for the service to be scalable, and also allows for easy updates with no downtime. For the database, I used AWS RDS. I utilised AWS VPCs so the RDS instance only allows incoming traffic from IPs in the same VPC, which would only be the ECS instances, ensuring the security of the data.

AI Image Classifier



By creating a range of models inluding MLP, CNN, and finetuned Transformers, we experimented with different methods of classifying images between the classes:

  • Real
  • Denoising Defusion GAN
  • Pallete
  • Synthetic Faces High Quality (SFHQ)
  • Style GAN 1

We trained our models off the CIFAKE dataset and our best model achieved an overall accuracy of 0.97. We utilised various techniques to optimise our models including dropout, GradCam, and class weights.

Light Trail Effect




Using the OpenCV library, I implemented this effect by having a frame that is continuously updated with each frame of the video. To have the lights linger, I had to devise a function to scale the amount each pixel gets updated depending on its intensity. Due to the computatioanlly intensive nature of this program, I implemented multi-core processesing, assigning each core of my CPU a different section of the image, then compiling the results into a final frame.

G-G chart




I implemented this by securing my phone to my bike, then using the accelerometer feature to gather some raw data. I then created an algorithm to find the bearing of the bike, as the phone would just be placed without any predefined position.
